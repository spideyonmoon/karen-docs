Get Karen up and running on your VPS in minutes.

## Prerequisites

- Linux VPS (Ubuntu/Debian) with Docker & Docker Compose
- Telegram Bot Token (from [@BotFather](https://t.me/BotFather))
- At least one Apple Music account (2FA disabled)
- (Optional) Telegram API credentials for MTProto uploads
- (Optional) Gofile API token for fallback delivery

## Get Started

<Steps>
  <Step title="Clone & Configure">
    ```bash
    git clone -b feat/wrapper-manager https://github.com/spideyonmoon/karen.git ~/karen
    cd ~/karen
    cp .env.example .env
    nano .env
    ```

    Fill in your secrets:

    ```bash
    TELEGRAM_BOT_TOKEN="YOUR_BOT_TOKEN"
    TELEGRAM_API_ID=12345
    TELEGRAM_API_HASH="YOUR_API_HASH"
    TELEGRAM_ALLOWED_CHAT_IDS="-100123456789"
    ADMIN_IDS="111111111"
    MEDIA_USER_TOKEN="YOUR_MEDIA_USER_TOKEN"

    APPLE_ID_1="user1@example.com"
    APPLE_PASS_1="password1"
    APPLE_ID_2="user2@example.com"
    APPLE_PASS_2="password2"
    ```

    <Tip>
      Add more `APPLE_ID_N` / `APPLE_PASS_N` pairs to scale horizontally.
    </Tip>
  </Step>

  <Step title="Bootstrap Services">
    ```bash
    ./setup.sh
    ```

    This runs one-time setup:
    1. Generates config from `.env`
    2. Builds `karen-wrapper-manager:local` image
    3. Boots all wrapper-manager instances
    4. Logs in each Apple account
    5. Starts the bot

    Monitor startup:
    ```bash
    docker compose logs -f bot
    ```

    Wait for: `Telegram bot started. Waiting for updates...`
  </Step>

  <Step title="Test It">
    Send your bot a Telegram message:

    ```
    /start
    ```

    Then rip a song:

    ```
    /dl https://music.apple.com/us/album/song-name/id1234567890?i=5678901234
    ```

    Watch the logs:
    ```bash
    docker compose logs -f bot
    ```
  </Step>
</Steps>
